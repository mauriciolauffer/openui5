Manual Integration Test

DESCRIPTION

Test the creation of new entities and verify whether the cancel and save functionalities work correctly.

PREPARATION

To check console errors open the developer tools before starting the application and check with each step, that there are no errors and warnings.

Testing with mock data

Start the application URL (adjust host and port if necessary):
  • local: http://<host>:<port>/test-resources/sap/ui/core/demokit/sample/common/index.html?component=odata.v4.Create

Testing with back end

Configure your IDE to be able to run the application against an ABAP back end system. Start the application URL (adjust host and port if necessary):
  • local: http://<host>:<port>/test-resources/sap/ui/core/demokit/sample/common/index.html?component=odata.v4.Create&realOData=true

REALIZE and CHECK

Start the application

  1. Check: The table is visible and contains several entries.
  2. Press the Create Sales Order button.
  3. Check: The table contains a new entry.
  4. Check: The dialog for creating a new entry opens.
  5. Set the Note by entering New note.
  6. Remove the focus from Note.
  7. Check: The field Note in the first entry of the table contains New note.
  8. Press the Cancel button.
  9. Check: The dialog for creating a new entry closes.
  10. Check: The new entry in the table disappears.
  11. Press the Create Sales Order button.
  12. Check: The table contains a new entry.
  13. Check: The dialog for creating a new entry opens.
  14. Remove the default value of 0100000000 for the Buyer ID.
  15. Press the Save button.
  16. Check: The dialog for creating a new entry is still open.
  17. Check: The Buyer ID field has the focus and the border is red, indicating an error.
  18. Set the Buyer ID by entering 0100000004.
  19. Remove the focus from Buyer ID.
  20. Check: The Buyer ID has the value state None.
  21. Remove the default value of EUR for the Currency.
  22. Press the Save button.
  23. Check: The dialog for creating a new entry is still open.
  24. Check: The Currency field has the focus and the border is red, indicating an error.
  25. Set the Currency by entering USD.
  26. Remove the focus from Currency.
  27. Check: The Currency has the value state None.
  28. Set the Note by entering New note.
  29. Press the Save button.
  30. Check: The dialog for creating a new entry closes.
  31. Check: The dialog with the success message opens.
  32. Press the OK button.
  33. Check: The dialog with the success message closes.
  34. Check: The first entry of the table contains
    • the entered data for the Buyer ID of the Buyer, Currency and Note.
    • the automatically provided data for the Company Name of the Buyer, Sales Order ID, Gross Amount, Lifecycle Status and Created At
