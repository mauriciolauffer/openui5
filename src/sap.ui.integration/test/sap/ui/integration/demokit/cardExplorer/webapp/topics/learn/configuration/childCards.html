Child Cards

Child cards enable the display of additional details from the parent card, provide additional views, actions, or navigation. They are defined in the "sap.card"/configuration/childCards property of the manifest and can be opened using the ShowCard action.

If card X needs to display its first child card Y, and card Y subsequently needs to display card Z, then the manifest for card Y must include the configuration for its child card Z.

Properties

Property Type Required Description Schema Version Since
manifest string Yes URL of the child card manifest. 1.79.0 1.141

Handling Destinations in Child Cards

Child cards must reuse destinations defined by their main card. This is achieved by setting the "sap.card"/configuration/useMainDestinations property to true and referencing the main card's destinations using the {{mainDestinations.destinationKey}} syntax.
For more information, see the Destinations topic and the Child Card Using Destinations sample.

Example

An example of a card with childCards:

{
	"sap.app": {
		"id": "card.explorer.actions.showHideCard",
		"type": "card",
		...
	},
	"sap.card": {
		"type": "Object",
		"configuration": {
			"childCards": {
				"detailsCard": {
					"manifest": "./detailsManifest.json"
				}
			}
		},
		"header": {
			"title": "Order for {ShipName}",
			...
		},
		"content": {
			"groups": [
				{
					"items": [
						{
							"label": "Name",
							"value": "{ShipName}"
						}
					]
				},
				...
			]
		},
		"footer": {
			"actionsStrip": [
				{
					"text": "Show More",
					"buttonType": "Accept",
					"actions": [
						{
							"type": "ShowCard",
							"parameters": {
								"childCardKey": "detailsCard"
							}
						}
					]
				}
			]
		}
	}
}
