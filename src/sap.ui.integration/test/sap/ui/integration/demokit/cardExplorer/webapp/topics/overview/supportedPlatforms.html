Supported Platforms

Web-Based Platforms

UI Integration Cards provide comprehensive support for web-based applications across desktop and mobile browsers.

Feature Support

  • Complete Feature Set: All card features are fully supported in web environments, including data binding, actions, filtering, and interactive elements
  • Responsive Design: Cards automatically adapt to different screen sizes and orientations.
  • Cross-Browser Compatibility: Consistent experience across modern browsers

Browser Requirements

For detailed browser support information, refer to the browser-support documentation of the OpenUI5 framework.

Mobile Native Platforms

True to the “Write once, use everywhere” principle, the UI Integration Cards are supported on mobile native platforms.
The mobile native implementation of UI Integration Cards features a built-in JavaScript engine capable of interpreting the same JSON manifest files used in web applications.
This embedded runtime processes the card definitions uniformly across platforms, ensuring consistent behavior and appearance.
As a result, developers can maintain a single card definition while achieving native performance and applying platform-specific optimizations on mobile devices.

What are Mobile Native Cards?

Mobile native cards are UI Integration Cards that run within native Android and iOS applications.

Supported Applications

  • SAP Mobile Start: Enterprise mobile portal that hosts and displays cards alongside other business content.
  • Joule for Mobile: AI-powered digital assistant that leverages cards for interactive business insights.

Feature Parity and Limitations

Most card properties are supported on mobile native platforms, with a few exceptions. These exceptions are marked throughout the documentation and fall into one of the following categories:

  • “Not applicable for mobile native cards.” - has no effect for mobile native integration cards.
  • “Not yet implemented for mobile native cards.” - this feature is currently not implemented, but might be implemented in the future.
  • “Not supported for mobile native cards.” - this feature can not be supported on mobile native platforms.

Additional Considerations for Mobile Native Platforms

  • Only declarative card types are supported. Cards of type Adaptive, Component and WebPage are not supported.
  • JS Extensions are generally supported, but they must adhere to the basic card facade API provided within the extension.
    The extension must be tested on a mobile native platform before it is productively released.
  • Certain web-specific APIs may not be available.
  • Some advanced animations or transitions may be simplified.
  • Network requests adhere to mobile application security guidelines.

Mobile native cards prioritize a native user experience while maintaining the core card functionality and consistent theming and design representation.
